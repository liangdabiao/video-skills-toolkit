# 公共素材库

跨项目复用的素材都放在这里。做视频时按需取用；项目里验证好用的新素材，交付后拷回这里登记，让库不断长大。

## 目录

```text
assets/library/
├── fonts/        # Studio 主题字体（脚手架自动播种进每个项目）
├── sfx/          # 通用音效（脚手架自动播种进每个项目）
├── music/        # 授权清楚、风格通用的 BGM 底（默认为空，按需取用）
└── animations/   # 可复用 Remotion 动效组件（按需拷进项目 src/library/）
```

## 复用判断原则（先想，再拷）

库是备选货架，不是必经流程。每次需要一个画面元素时，按这个顺序判断，三条路都是正当的：

1. **先问"这条视频需要什么"**，再看库里有没有，而不是反过来拿库里有的东西凑画面。
2. **复用前核对匹配度**：这个组件的视觉调性（颜色/线条/密度）、动效节奏、信息结构是否符合当前视频的设计系统和这一幕的叙事任务？只是"功能上像"不够——聊天窗组件是 Studio 暖白风，这条片是深色故事片，就不该直接拿来用。
3. **改造或新写都不算浪费**：改造成本低就 fork 一份改；场景差得远就直接新写一个更贴的。新写的东西验证好用后再考虑回流进库。
4. 分镜或代码里用了库组件，要能说出"为什么它比新写更合适"；说不出来就重新想。

## 使用方式

- **fonts / sfx**：脚手架自动复制到项目 `public/assets/fonts/` 和 `public/assets/audio/`，不用手动管。
- **animations**：需要哪个动效，就把那个 `.tsx` 文件拷进项目的 `src/library/`，再 import 使用（类似 shadcn 的复制模式，项目保持独立）。
- **music**：需要时把文件拷进项目 `public/assets/music/`。

## 沉淀新素材的规则

1. 新素材先在具体项目里做出来、用过、效果好（或用户明确选中）之后，才拷回库里。实验性片段不进库。
2. 动效组件必须自包含：单个 `.tsx` 文件、只依赖 `remotion` 和 React、props 有默认值、文件头部用注释写清用法。
3. 每个新素材都要在下面对应清单里加一行，写清"适合什么"和"不适合什么"。
4. 音频类素材必须在同目录 `SOURCES.md` 里补来源/授权说明，没有来源的只能当草稿。

## 动效清单（animations/）

| 组件 | 用途 | 适合 | 不适合 | SFX 卡点 |
|---|---|---|---|---|
| `TitleSlam.tsx` | 大字逐个砸落进场，落点带整屏震动 | 开场 Hook、章节大标题、重大反差结论（2-6 个字） | 普通关键词出现、长句、安静解释段 | 每字落点帧见文件头注释；通常首/末落点放重型 impact |
| `ChatWindow.tsx` | 仿真 AI 对话窗，消息气泡按字幕 cue 弹入 | AI 对话、多轮提示词迭代、提问与回答对比 | 长篇逐字稿、需要真实产品截图证明的步骤 | 每条气泡 appearAt 帧适合轻 pop |
| `Terminal.tsx` | 仿真终端，命令逐字符打出、输出逐行出现 | 安装依赖、命令执行、脚本跑起来、错误输出 | 大段代码讲解 | 命令回车帧、error 行出现帧 |
| `CodeEditor.tsx` | 仿真代码/提示词编辑器，重点行按 cue 扫入高亮 | 讲代码、提示词、配置片段中的关键行 | 命令执行过程、超过 15 行的长文件 | 每次高亮扫入帧适合轻 sweep |
| `ErrorPopup.tsx` | 翻车/警告弹窗层，进场震动并持续辉光 | 报错、风险、警告、上下文超限等情绪点 | 承载整屏内容或常驻信息面板 | appearAt 落地帧可配 `sfx-deep-impact.mp3` 或轻量变体 |
| `FlowDiagram.tsx` | 节点 + 连线逐段生长的示意动画 | 流程、管线、因果链、三五个节点的架构关系 | 复杂自动排版图、长文本节点 | 节点落位帧轻 pop；关键连线长完帧轻 sweep |
| `StatCounter.tsx` | 数字滚动和横向数据条推进 | 倍数、比例、耗时、返工率、前后对比 | 没有数据含义的装饰性大数字 | 数字落定帧、每条 bar 到位帧 |
| `ArticleImage.tsx` | 公众号/文章原文图完整展示 + 标题/解读/图源 overlay，**`object-fit: contain` 永不裁切** | 公众号截图、表格、流程图、信息图等带文字的图 | 装饰性背景图、高分辨率摄影图、纯插画 | 图进场帧轻 pop，图源标签 6s 呼吸不抢戏 |

## 待沉淀组件（优先级清单）

当前优先级清单已实现并登记到上方动效清单。后续新增候选组件时，先写进这里和 `animations/COMPONENT-SPECS.md`，验证后再移入正式清单。**注意：它们始终是备选货架，不是默认答案**——每次用之前按上面的「复用判断原则」核对调性和场景，不合适就改造或新写。

已实现组件的原始规格归档在 [animations/COMPONENT-SPECS.md](animations/COMPONENT-SPECS.md)。

## 音效清单（sfx/）

详细来源见 `sfx/SOURCES.md`。选用参考：

| 类型 | 文件 | 适合 | 不适合 |
|---|---|---|---|
| 重型开场冲击 | `sfx-deep-impact.mp3`、`sfx-movie-trailer-epic-impact.mp3` | 开头 Hook、大标题落点、重大反差 | 普通关键词、轻微 UI 动效 |
| 快速转场 whoosh | `sfx-whoosh-fast-transition.mp3`、`sfx-small-sweep-transition.mp3` | 场景切换、章节推进、截图扫入 | 小行文字出现、安静解释段 |
| 关键词/卡片 pop | `sfx-hard-pop-click.mp3`、`sfx-long-pop.mp3` | 关键词出现、卡片落位、列表行出现 | 每一条字幕都加 |
| 确认 beep | `sfx-positive-interface-beep.mp3` | 选项选中、进度确认、成功节点 | 严肃警告、大情绪点 |

## 音乐清单（music/）

暂无。沉淀标准：纯音乐、授权清楚、风格通用、人声友好（中频留空间、无突然 drop）。
