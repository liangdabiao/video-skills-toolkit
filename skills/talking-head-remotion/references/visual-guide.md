# Studio 视觉规范

Remotion 口播视频的默认设计语言。写或改 composition、选颜色字体布局前读这里。

## 视觉核心

现代编辑部式 Studio 风格：暖白全画幅画布、上下双层镜像透视格子、黑灰文字、克制蓝色强调、大留白、细线、右下角紧凑口播 PIP、底部无底色黑字字幕。**默认画幅 16:9 横屏：1920x1080 / 30fps**。本文的所有固定层尺寸（顶栏高度、安全区 padding、PIP 位置、字幕位置）都是横屏值；做 9:16 竖屏时这些层需要重新设计，不能直接套用。

## 字体

- 中文优先 `Noto Sans SC`，英文和数字优先 `Space Grotesk`；允许用稳定系统字体栈兜底，避免渲染因字体加载卡死。
- 在 `src/theme.ts` 用本地 `@font-face` 非阻塞注册 `public/assets/fonts` 里的字体，或使用系统字体栈。不要默认用 `@remotion/fonts/loadFont()` 阻塞渲染；它会触发 `delayRender()`，长视频可能在任意帧等待字体超时。
- 除非用户明确要求换主题，只用这套 Studio 字体栈。

## 固定界面层

### 舞台背景

- 全屏 `PremiumGridBackground`：动态淡色 wash、顶部透视格子、底部镜像透视格子、上下对称 horizon line、中心 veil 和轻微 grain。
- 顶部格子像天花板，底部格子像地面；底部必须复用上层行距、投影函数、纵线端点、线宽和透明度，只通过 `mirrorY = 1080 - y` 镜像，不能另起一套格子。
- `PremiumGridBackground` 必须实际调用 `PerspectiveGrid`；代码里声明了但没调用视为失败。
- 不要换成纯白、灰色、暗色赛博、全屏渐变、普通 CSS 平面网格；不要把内容包进白色圆角卡片、缩小画板、网页 preview 容器或 dashboard shell。

### 顶部进度条

- 高 `68px`，全宽，背景 `#202024`；白色进度填充 `top:7px; bottom:7px; left:0; border-radius:0 14px 14px 0`。
- 章节行 `display:flex; justify-content:space-between; padding:0 34px; font-size:22px`。
- 标签面向观众（不写内部 scene id），最多 5-7 个，位置匹配字幕同步后的真实章节起点。

### 主内容安全区

- `padding: 196px 120px 196px`。
- **文字宁少勿多，甚至可以无字**：主画面文字只允许关键词、仓库/项目名、数字、短对比词、短 CTA，完整口播句子只放字幕。禁止把逐字稿内容搬上主画面——历史教训是"满屏都是字"。一屏同时可见的主画面文字元素不超过 5 个。
- **画面丰富度靠非文字层做，不靠加字**：砍掉文字后空出来的信息量，用图示、仿真 UI、示意动画、动态图形补上（见"画面丰富度硬规则"）。
- 用细线组织版面，不要堆卡片（除非卡片本身就是内容对象）。

### 画面丰富度硬规则

和"禁止 PPT 式整页出现"同等强度，违反视为失败：

1. **每屏至少一个非文字视觉主体**：图示（流程/节点/层级图）、仿真 UI（聊天窗、终端、代码编辑器、报错弹窗）、录屏/截图、数据可视化（进度条、计数器、对比条）、或纯图形动画。只有关键词大字 + 细线的场景，整条视频最多出现 2 次（如封面、CTA）。
2. **凡讲到能截图/能录屏的具体东西**（产品操作、AI 对话、代码在跑、报错），必须给真实或仿真界面，不能用关键词或隐喻图形代过（经验 L1）。没有真实素材就用组件仿真。
3. **任一时刻画面至少有一层持续动效在动**（背景 wash 流动、图示连线生长、数据推进、光标移动、辉光呼吸等，幅度可以很轻）。"口播解释时画面可以停住"指的是不加新的进场，不是全画面静帧。
4. **同一版式连续不得超过 2 个场景**（经验 L2）：连续场景要换布局模式、换视觉主体类型或加一层新的动态素材，让观众能察觉"进入新内容了"。
5. 抽象设计语言（隐喻图形、色彩系统）继续作为骨架保留；具象是证据，抽象是骨架，两者搭配（经验 L1 的边界）。

### 口播 PIP

- 右下角圆形，默认 `206px`，位于 `right:104px; bottom:150px`，显示头和肩膀。
- 外圈 `inset:-11px; border:1.5px solid rgba(20,82,255,0.55)`，白色内边距 `5px`，柔和阴影 `0 16px 40px rgba(0,0,0,0.14)`。
- 口播视频用 `<Video muted>`；没有视频时用中性占位。避免挡字幕。

### 字幕

- 相对整个画布居中，底部 `96px`；无背景底色、无圆角框，直接黑字 `#151922`，字号 `38-40px`、字重 500。
- 承载完整口播句子，同一时间只显示一组；用蓝色 `#2f6fff` 加粗强调一两个关键词。
- 避开右下角 PIP。

## 布局模式库

- 封面/标题：眉标线 + 期数/标签 + 大标题 + 弱副标题。
- 标准/列表：左侧标题栏 + 右侧三到五行稀疏编号列表和细线。
- 数据/数字：巨型数字或短指标 + 右侧说明和小数据行。
- 对比/选择：两栏，中间单条细线；一侧可以用蓝色强调。
- 结尾/CTA：居中短标题 + 弱提示行。
- 证据面：大截图或录屏，用细线框住，不要做厚重卡片。
- 仿真 UI 面：组件仿真的聊天窗 / 终端 / 代码编辑器 / 报错弹窗做视觉主体，内容按字幕 cue 逐条弹入或逐行高亮。
- 示意动画面：无字或极少字，用图形运动本身讲逻辑（流程连线生长、方块堆叠、卡片飞入、数据流动），关键词最多作为锚点出现。

选择能服务口播观点的模式；纯文字类模式（封面/标题、标准/列表）整条视频合计不要超过一半场景。

## 动效与节奏

- 所有动画由 Remotion 帧驱动：`useCurrentFrame()`、`interpolate()`、`Easing.bezier()`、`Sequence`（设置 `premountFor`）。不要用 CSS animation、CSS transition 或浏览器计时器。

### 动效词汇表（四类都要用，不能只有进场）

历史毛病是"动画=进场动画"，元素蹦出来之后整屏静止十几秒。每条视频四类动效都要出现：

| 类型 | 说明 | 帧驱动实现范式 |
|---|---|---|
| **进场动效** | 元素首次出现，0.25-0.45 秒，位移+透明度+轻微缩放 | `interpolate(frame, [cue, cue+12], ...)` + spring |
| **持续动效** | 元素在场期间的低幅度运动：辉光呼吸、连线流光、光标闪烁、背景 wash 漂移、数字滚动余韵 | `Math.sin(frame / 周期)` 做循环；幅度小到不抢字幕注意力 |
| **强调动效** | 口播再次提到已在场元素时的反应：放大回弹、变色、圈注、震动一下 | 给元素加 `emphasizeAt` cue，在该帧触发短促 spring |
| **示意动画** | 用运动本身讲逻辑：流程连线逐段生长、数据条推进、消息气泡逐条弹入聊天窗、代码逐行高亮、光标点击按钮 | 多个子元素各带 cue，按口播顺序串成一段"会讲话的动画" |

- 抽象概念优先用**示意动画**表达而不是关键词大字：说"上下文越堆越多"就让方块真的堆起来，说"两个提示词"就让两张卡片真的飞进对话框。让画面演出来，字只做锚点甚至不出现。
- 强调动效和示意动画的关键帧要在 SFX cue sheet 里留卡点。

### 逐元素进场（核心节奏规则）

- **禁止 PPT 式整页出现**。场景切换只是换舞台；场景里的每个可见元素都要有自己的进场时刻和进场动画，不能在场景第 0 帧全部铺好。
- 进场顺序和时刻由字幕驱动：给每个元素定一个 cue 时间（秒），来自"口播说到这个内容"的那条字幕（或字幕里的那个词）的起点。列表逐行、指标逐个、对比先 A 后 B，都对齐各自的口播短语。
- 数据结构上，给场景里的每个元素带上 `appearAt`（场景内相对秒数）字段，而不是整个场景共用一个开始时间。模板的 list items、stat metrics、compare choices 已支持可选 `appearAt`；新场景类型也按这个模式实现。
- 进场动画要短而明确（0.25-0.45 秒），用位移 + 透明度 + 轻微缩放的组合；同一场景内相邻元素至少错开 0.12 秒，避免同时蹦出。
- 元素之间的留白也是节奏：口播在解释时不加新的**进场**，但持续动效保持在动（见丰富度硬规则第 3 条），不允许全画面静帧。
- 元素在场景结束前保持在场，不要出现后又无故消失。

### 其他

- Hook 可以在时间上有冲击，但视觉上保持 Studio 气质：干净、编辑感、克制。
- 给 SFX 预留清楚的视觉冲击帧（标题落点、关键词 pop、场景扫入、指标落位、CTA 落点），不要留模糊的动画时间。

## 交付前抽帧检查

至少抽 Hook、章节切换、数据页、CTA 的帧，确认：

- 背景接近 `#f7f8f3`，全画幅，上下格子镜像一致（同 row step、投影函数、纵线端点、线宽、透明度）。
- 没有白色圆角容器、缩小卡片或网页 shell 包住画面。
- 顶栏、字幕、PIP 在固定位置；字幕没有和 PIP 或画面内容冲突。
- 主画面只有关键词或短语（≤5 个文字元素），没有整句逐字稿；字体栈符合 Studio 主题，且没有字体加载超时。
- 对多元素场景抽"场景开始后 0.3 秒"和"场景中段"两帧对比：前一帧元素应该还没出全，逐个进场才算合格；两帧一模一样就是 PPT 式出现，视为失败。
- **丰富度检查**：每个抽到的场景至少有一个非文字视觉主体；在场景中段间隔 1 秒抽两帧对比，两帧完全相同说明没有持续动效，视为失败。
- **版式重复检查**：把各章节代表帧并排看，连续 3 个场景版式和视觉主体类型都相同视为失败。
