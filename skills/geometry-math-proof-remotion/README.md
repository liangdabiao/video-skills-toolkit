# Geometry Math Proof Remotion · 精准几何证明视频

把数学证明文档变成深底精准几何风格的 Remotion 视频。几何全部代码绘制（SVG + stroke-dash 描边），公式逐行揭示，字幕驱动时间轴。3Blue1Brown / 可汗学院风格。

> 证明视频的灵魂不是酷炫的转场，而是推导的逻辑链条。跳一步，观众就跟丢了。

---

## 这是什么

一套数学证明 / 几何动画视频的 Remotion 工程模板，适合做：

- 📐 几何定理证明（勾股定理、相似三角形、圆周角…）
- 🔢 代数恒等式推导（二项式定理、欧拉公式…）
- 📊 微积分可视化（积分、导数、极限…）
- 💻 算法动画（排序、搜索、数据结构…）
- 🧮 任何需要"一步步推导给人看"的数学内容

## 核心原理：字幕驱动 + 逐步揭示

很多数学动画的问题是：画面很漂亮，但节奏和讲解对不上。要么公式堆在一起看不过来，要么一个式子停十秒无聊死。

**正确做法：**

1. **先写逐字稿，生成 TTS 音频，拿到字幕** —— 音频时长决定一切
2. **把每一步推导对齐到对应字幕的时间点** —— 说到哪一步，画面就揭示到哪一步
3. **公式面板从右往左逐步叠** —— 旧的往上收，新的从底部进来，观众始终知道"我们推到哪了"

```
逐字稿 → TTS → 字幕 → F 时间轴对象 → 公式逐步揭示 → 几何描边动画
              ↑
          唯一的时间基准
```

## 证明的五段式结构

每个证明都拆成 6~10 个章节，遵循同一条叙事线：

```
钩子(H)        → 抛出命题，大字公式 + 悬念，抓住注意力
准备(P)        → 介绍构造，画三角形 / 引入变量 a b c
推导 1 ~ N     → 每章只证一件事：局部面积、整体面积、一次化简
消项/合成(E)   → 把局部结果拼成最终等式，黄框大字
收尾(F)        → Q.E.D. + 历史人物 / 年代署名
```

**为什么要拆这么细？** 因为人理解数学的速度比你想的慢。一个"显然"可能需要 3 秒才能反应过来。每章只推一步，观众才能跟上。

## 风格 DNA（不可变）

| 项 | 值 |
|---|---|
| 画布 | 1920×1080 @ 30fps，深底 `#0d0d12` |
| 配色 | 红 `#e74c3c` / 蓝 `#3498db` / 绿 `#2ecc71` / 黄 `#f1c40f`（黄=最终结论/钩子） |
| 字体 | 数学衬线 `Cambria Math` / `STIX Two Math`，中文 `PingFang SC` |
| 几何 | SVG viewBox `0 0 1100 860`，单位 SCALE=130px |
| 描边 | 一切线框用 `strokeDasharray` + `strokeDashoffset` 做"被画出来"动画 |
| 动画 | `fadeIn` / `drawIn` / `fadeOut`，缓动统一 `bezier(0.4,0,0.2,1)` |
| 字幕 | 底部 110px 字幕条，关键词 `tone:'accent'` 黄色加粗高亮 |
| 公式 | 右侧 700×820 面板，逐步揭示；最终结论 `tone:'result'` 黄框大字 |
| 素材 | **全部代码绘制**，不用任何位图 / 图标库 / emoji |

## 关键帧节奏

一个元素的完整出场：

```
描边渐进（36帧）→ 填充淡入（30帧）→ 标签淡入（24帧）
```

一个公式步骤淡入：24 帧

**记住：** 章节末 30 帧不要再上新元素。给观众留消化时间。

## 证明忠实度原则

- 源文档是证明，不是提纲。可以合并句子，但不能跳步导致推导链条断裂
- 保留"承接步骤"——不能直接说"面积是 c²/2"，必须先有"两腰都是 c"做铺垫
- 旁白可以精简，但必须保留因果推进：构造 → 局部 → 整体 → 消项 → 结论
- TTS 前对照 beat checklist 逐条检查，确认没漏关键定义 / 构造 / 消项步骤
