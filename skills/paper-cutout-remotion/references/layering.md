# 四层拆分方法论（纸片风的灵魂）

> 真正的纸片分层动画，核心不是生成一张漂亮的图，而是把背景、建筑、主角、配角和前景
> 分别做成独立素材，再让每一层按照不同节奏运动。
> —— 需求原文

这是纸片风区别于"一张图加镜头移动"的根本所在。本文件是这套方法论的落地参考。

## 一、为什么必须拆层

如果只把一张完整图片放大缩小 + 镜头移动，画面能动，但**没有层次**。

拆层后能做到：
- 主角独立飞入、配角错峰补充、群像逐步建立
- 每层按不同节奏运动（背景慢推、后排微动、主体有力入场、前景稍快）
- 前后遮挡制造纵深立体感（不用真的做 3D）
- 同一套模板复用于不同题材（历史/科普/商业/人物关系）

## 二、四层模型

每个镜头至少拆成下面四层。从后到前的渲染顺序 = zIndex 从小到大：

| 层级 | zIndex | 内容 | 动画特点 | role |
|---|---|---|---|---|
| 背景层 | 0 | 山水、宫殿、城楼、纸张纹理 | 最慢，只做轻微漂移和推镜（BackgroundPan 1-3%） | — |
| 后排层 | 1-2 | 远处群臣、侍从、建筑边角 | 移动幅度小，可略降饱和度/透明度 | `tertiary` |
| 主体层 | 5 | 皇帝、中央礼盒人物 | 尺寸最大，入场最有力量，缩放最狠 | `primary` |
| 前景层 | 3-4 | 侍女、跪拜人物、纸屑、胶带 | 移动稍快，用来遮挡并制造纵深 | `secondary` |
| 文字/字幕 | 6+ | 标题、章节标签、字幕 | 最上层，不参与遮挡 | — |

⚠️ **注意前景层 zIndex（3-4）小于主体层（5）**——前景在主体"前面"但 zIndex 更小？
不，zIndex 越大越靠前（视觉上越上层）。所以前景层应该 zIndex 更大才对。

**修正**：前景层的作用是**遮挡主体**制造纵深，所以它在视觉上更靠前 → zIndex 应该**大于**主体。
正确分配见下表（这是最终规范）：

| 层级 | zIndex | role | 说明 |
|---|---|---|---|
| 背景层 | 0 | — | Plate 底板 + BackgroundPan |
| 后排层 | 1-2 | `tertiary` | 远处人物，最弱存在感 |
| 主体层 | 3-4 | `primary` | 主角，但被前景局部遮挡增加纵深 |
| 前景层 | 5-6 | `secondary` | 近处人物/装饰，遮挡主体边缘 |
| 文字/字幕 | 10+ | — | 永远最上层 |

> **设计要点**：纸片风的立体感来自**遮挡关系**，不是 3D 透视。前景故意挡住主体的脚/边缘，
> 观众的大脑会自动脑补出空间前后。所以前景 zIndex 必须大于主体。

## 三、先定镜头，再画图（顺序铁律）

**不要先画图再排版。** 同一个人物在全景和特写里的大小、位置、朝向都不同。
先随便生成素材，后面必然出现：人物比例错误、视线方向相反、主次不清。

每个镜头先回答三个问题：
1. **谁最大？**（主角，决定观众先看哪）
2. **配角在哪？**（不能挡主角的脸/手/关键道具）
3. **地平线在哪？**（所有人物脚底落在同一个合理地面上）

### 全景镜头 vs 特写镜头

| 镜头类型 | 目的 | 主次安排 |
|---|---|---|
| 全景 | 交代规模（"盛唐长安"） | 皇帝最大居中，左右侍女次之，右侧群臣最小 |
| 特写 | 表现情绪/动作（"万邦来朝"） | 中央捧礼盒人物最大，前排跪拜次之，后排侍从最小 |

## 四、角色宽度参考（唐朝片实测值，排版锚点）

不是硬约束，是"主次大小要拉开差距"的量级参考。所有值都是 Remotion 里的 `width` px：

| 角色 | 全景镜头 | 特写镜头 |
|---|---|---|
| primary（主角） | 650 | 575 |
| secondary（次要） | 235-245 | 285-330 |
| tertiary（后排） | 158-170 | 165-180 |

关键观察：**主角是后排的 4 倍宽**。如果主次差距不到 2 倍，画面会显得平均、没重点。

（这些值已存进 `cutout.tsx` 的 `WIDTH_REF` 常量，直接引用即可。）

## 五、错峰入场（不要同时出现）

人物同时入场 = 死板。用 `delay` 错开：

```tsx
// 主角先出（delay 4），配角补充（delay 16-22），后排最后填充（delay 30-40）
<PaperActor src="emperor.png" role="primary" delay={4} ... />
<PaperActor src="maid-left.png" role="secondary" delay={18} ... />
<PaperActor src="minister.png" role="tertiary" delay={34} ... />
```

画面按"主角先出现 → 配角补充 → 群像完成"的顺序建立，而不是一下子塞满。

## 六、白边 + 投影（把人物从背景抬起来）

`cutout.tsx` 的 `PAPER_FILTER` 配方：

```css
filter:
  drop-shadow(4px 0 #f5eedc)      /* 右白边 */
  drop-shadow(-4px 0 #f5eedc)     /* 左白边 */
  drop-shadow(0 4px #f5eedc)      /* 下白边 */
  drop-shadow(0 18px 9px rgba(20,15,12,.32));  /* 深投影 */
```

- **白边**（前三向）：负责"剪纸感"，让人物轮廓清晰，不糊进背景
- **深投影**（第四向）：负责把人物从背景"抬起来"，制造悬浮立体感

四项缺一不可。少了白边会糊，少了投影会扁。

## 七、极轻微漂浮（避免画面静止）

所有角色入场后仍保留极轻微上下漂浮（`bob`），避免画面完全静止显得假：

```tsx
// cutout.tsx 内部实现
const bob = Math.sin(frame * 0.06 + randomPhase) * roleMotion[role].bobAmp;
```

不同角色用不同 `seed` → 不同相位 → 不会同步漂浮（同步会露馅）。

## 八、静态排版检查清单（加动画前必做）

拿到拆好的独立 PNG 后，**先不加动画，静态摆好**，检查：

- [ ] 主角是不是最大、最醒目？
- [ ] 配角有没有挡住主角的脸、手、关键道具？
- [ ] 所有人物脚底是不是落在同一个合理地面上？
- [ ] 朝向是否符合场景关系（朝向主角 = 对话/朝拜；背对 = 离开）？
- [ ] 前景遮挡是否制造了纵深（而不是误挡主体关键部位）？

全部通过后，再按 role 加错峰动画。这一步决定了画面是否自然。
